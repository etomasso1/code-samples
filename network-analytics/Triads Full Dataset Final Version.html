# Define the triad function used later on for identifying triads and potential triads

def triads(m):
    out = {0: set(), 1: set(), 2: set(), 3: set()}
    nodes = list(m.keys())
    for i, (n1, row) in enumerate(m.items()):
        #print(f"--> Row {i + 1} of {len(m.items())} <--")
        # get all the connected nodes = existing keys
        for n2 in row.keys():
            # iterate over row of connected node
            for n3 in m[n2]:
                # n1 exists in this row, all 3 nodes are connected to each other = type 3
                if n3 in row:
                    if len({n1, n2, n3}) == 3:
                        t = tuple(sorted((n1, n2, n3)))
                        out[3].add(t)
                # n2 is connected to n1 and n3 but not n1 to n3 = type 2
                else:
                    if len({n1, n2, n3}) == 3:
                        t = tuple(sorted((n1, n2, n3)))
                        out[2].add(t)
        
    return out

"""
The triad function above is used to find completed triads and potential triads : triads missing one leg of the triangle

The code in the cell below is run for each progressive month after month 1, where the number of triads 
and potential triads are counted. With more time, we would have compared the completed triads in month x
with the potential triads in all previous months to find the probability of closure. We also would have integrated
the count function to assess the probability, but did not have time as we only found this function on Tuesday.

The cell below works by changing n to the month you want to analyze, and all months up to that point are included. 
Once the code is executed, there are a series of cells below which contain len() functions for each period. This was
a simple way to measure the change over time, although it is not the most elegant and requires some manual work.

However, this method allowed us to analyze a far longer period of time than with the Kossinets Watts model. 
The other change we made was to take the value counts for each month as this significantly compressed the dataset
and allowed us to analyze a longer time period. 

In addition, we installed a swapfile on the AWS of 10 gb which helped process larger data slices.
"""
